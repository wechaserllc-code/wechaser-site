03 / 内容质检Content QA

不只是"扫一遍 OCR 完事"。 It's not just "OCR and done".

四类常见错误必查——这些都是我们在轨交客户实战中踩过的坑。 Four common error classes are explicitly audited — landmines we've stepped on in production.
01 / OCR DRIFT

OCR 字符漂移OCR character drift

"0/O" "1/l" "B/8" 等容易识别错。我们对关键字段（型号、参数值、料号）做正则模式校验 + 二次模型 cross-check。

"0/O" "1/l" "B/8" classic confusions. Critical fields (part number, parameter value, SKU) get regex pattern validation + secondary-model cross-check.

02 / TABLE BREAK

表格断裂Table fragmentation

跨页表 / 合并单元格容易丢行。MinerU 解析 + 行数校验，对不上数会标红。

Cross-page tables / merged cells often drop rows. MinerU parse + row-count verification — mismatches flagged.

03 / DUPLICATE

同源重复Same-source duplicates

同一份文档 v1/v2/v3 散落在不同文件夹。file_hash 唯一约束 + 内容相似度双层去重，保留最新版本。

v1/v2/v3 of the same doc scattered across folders. Two-layer dedup: file_hash uniqueness + content similarity. Keep the latest.

04 / SEMANTIC

语义边界混淆Semantic boundary mix-up

"A 产品规格"被解析成"B 产品规格"——这种错误最危险。靠段落上下文 + 关键词锚点双验证，可疑段落人工复查。

"Product A spec" parsed as "Product B spec" — most dangerous. Detected via paragraph context + keyword anchor double-check; suspicious paragraphs go to human review.
