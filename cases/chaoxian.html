01 / 背景Background

为什么是超弦。 Why Chaostring.

选客户和选 co-founder 一样重要——他们的需求会塑造产品。 Choosing your seed client is as load-bearing as choosing a co-founder — their needs shape the product.

超弦做的不是"小硬件"——是铁路机车的控制系统板卡。一块板要进入实际机车运行，意味着每一份图纸、每一份工艺、每一个元器件型号都得经得起运行追溯。

Chaostring doesn't ship "small hardware" — they ship control-system boards for railway locomotives. For a board to enter actual rolling stock, every drawing, every SOP, every component spec must survive operational traceability.

这种业务对幻觉零容忍——AI 在这种环境下不能"差不多"。所以一开始就以这套系统为标准做时光脉络，比从消费 AI 起步要难，但能站得住。

This kind of business has zero tolerance for hallucination — "close enough" is not an answer. So building Shiguangmailuo against this bar from day one is harder than starting from consumer AI, but it's the only way to stand on solid ground.

另一个关键：王总（决策人）愿意给真实业务文档做样本，并在每次咨询后给出反馈。这是工业 RAG 系统能跑起来的核心条件。

Equally important: Mr Wang (the decision-maker) agreed to share real internal docs and give feedback after each engagement. That's the prerequisite for any industrial RAG system to actually function.

04 / 得到的教训Lessons

客户教我们的事，
比我们写的代码更重要。 The lessons from this client
matter more than the code we shipped.

LESSON 01

Component 不只是"小零件"Component ≠ "small part"

我们一开始以为 Component 是 AGV 的传感器之类——错了。在超弦语境里，Component 是铁路机车控制系统的电子设备（TAX07/LKJ 数据转存器/RK3588 等）。语义错位会导致整个图谱设计错。

We initially modelled Component as something like AGV sensors. Wrong. In Chaostring's world a Component is a railway-locomotive control electronic device (TAX07, LKJ data recorder, RK3588…). A semantic mismatch this big invalidates the whole graph schema.

LESSON 02

先让能用，再谈花哨Run first, polish later

客户王总反复强调："先让能简单安装、操作。能用起来，边用边改。" 不优先：花哨 UI / 新功能 / 未来场景。优先：macOS 打包 / 一键安装 / 有 app 图标 / WebUI。

Mr Wang's repeated principle: "make it installable and usable first; iterate as we use it." De-prioritized: fancy UI / new features / future scenarios. Prioritized: macOS packaging, one-click install, app icon, WebUI.

LESSON 03

飞书是真接口，不是市场宣传Lark is the real interface

制造业内部沟通工具是飞书——所有 AI 应用必须先在飞书里跑通，否则员工不会用。我们已经把 Hermes Gateway 的飞书适配器接通，下一步是 Bot 维度的真实部署。

Manufacturing's internal collaboration is Lark — any AI app must work inside Lark first or employees won't use it. The Hermes Gateway Lark adapter is wired; next is bot-level production deployment.

LESSON 04

咨询是入口，知识库是产品Consulting is the door, KB is the product

商业模式（特洛伊木马）经实测有效：高价咨询打开门 → 送知识库 → 客户用起来后主动找我们做更多。我们做了 2 次咨询，客户主动提需求 1 次——比"卖产品"的转化率高得多。

The Trojan-horse model is validated: paid consulting opens the door → free knowledge-base hand-off → client uses it → client comes back asking for more. After two consulting rounds, the client has already proactively asked for additional capability — a higher hit rate than direct sales.
